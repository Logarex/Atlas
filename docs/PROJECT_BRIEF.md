# Project Brief

## Working Name

Atlas.

The final public app name should stay neutral and descriptive. Avoid names that imply endorsement by Apple, and avoid cloning other proprietary apps branding.

## Vision

Build the most complete, carefully sourced, community-maintained atlas of Apple retail stores worldwide.

The app should feel premium and calm, but the value is trust: complete coverage, sources, history, map discovery, bilingual access, and a contribution system that keeps bad data out.

## Audience

- Apple retail architecture enthusiasts.
- Travelers who visit notable stores.
- People tracking stores they have seen.
- Contributors who know local stores, remodels, closures, and photos.
- Researchers interested in retail design history.

## Core Product

- Global directory of stores, including open, closed, relocated, temporary, and announced stores.
- Store detail page with name, location, status, opening/closing history, source links, photos, and architectural attributes.
- Map with store markers, clustering, filtering, and nearby discovery.
- English and French interface.
- In-app data corrections and photo submissions.
- Moderation through GitHub issues.
- Local user profile with username, visited stores, and visit dates.
- Native share card for Instagram Stories and the system share sheet.

## Store Attributes

Initial attribute groups:

- Identity: store id, display name, country, city, store number, official URL.
- Lifecycle: announced, opened, relocated, remodeled, temporarily closed, permanently closed.
- Location: address, coordinates, geohash, timezone.
- Architecture: era, typology, facade material, architect, boardroom, green wall, forum, video wall, plaza, stair type, mall/standalone/flagship.
- Operations: regular hours, special hours, last verified date, official hours URL.
- Community: photos, visit count, contributor credits, confidence score.
- Provenance: field-level source links, license, reviewer, verification date.

## MVP

The first credible public release should include:

- Bilingual app shell.
- Store list, search, detail pages, and map.
- Seed dataset for a small number of well-sourced stores.
- Local username profile.
- Mark as visited with optional date.
- Submit correction from the app.
- GitHub issue creation for new submissions.
- Maintainer approval through dataset pull requests.
- Privacy page and detailed README.

## Later

- Full worldwide dataset.
- Closed and relocated store history.
- Photo moderation and contributor galleries.
- Instagram Story share cards.
- Offline cache.
- Import pipeline from compatible open data.
- Public API for the curated dataset.
